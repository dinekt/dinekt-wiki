# マルチエージェントの8つの失敗モード

### 1. Context Compression Amnesia
コンテキスト圧縮時にエージェントが重要な知識を忘れる。
- **症状**: 前のターンで決めた方針を圧縮後に覆す
- **対策**: MEMORY.md / proposals/ で重要決定を外部永続化。ADRパターンも有効

### 2. Self-Review Blindness
同一モデルが自己レビューするとバイアスが残る。
- **対策**: 批判者エージェント（別ロール）による独立評価
- **Dinekt**: 対策済み（critic/）

### 3. Mid-Task Agent Failure
長時間タスクで途中死亡 → 中途半端な状態で残る。
- **症状**: バックグラウンドエージェントがネットワークエラーやトークン制限で停止
- **対策**: チェックポイント戦略（後述）
- **Dinekt**: 対策追加（本ドキュメントで実装）

### 4. Concurrent File Edit Conflicts
複数エージェントが同じファイルを同時編集して競合。
- **症状**: Worktree並列時にマージコンフリクト多発
- **対策**: ファイル所有権の事前宣言ルール（後述）
- **Dinekt**: 対策追加（本ドキュメントで実装）

### 5. Flat Task List Inflexibility
フラットなTODOで回復不可、ゲートなし。
- **対策**: フェーズゲート付きタスク管理
- **Dinekt**: secretary/todos/ で緩和済み

### 6. Lost Cross-Session Task Memory
セッション間でタスクの記憶が消える。
- **対策**: MEMORY.md + secretary/todos/ で永続化
- **Dinekt**: 対策済み

### 7. Knowledge Death Between Sessions
意思決定の経緯が次セッションで消える。
- **対策**: ADRパターン（docs/decisions/）
- **Dinekt**: 対策済み（ADR導入完了）

### 8. Token Waste During Idleness
低価値操作（同じファイルを何度も読む、無意味な検索を繰り返す等）でトークンを浪費。
- **対策**: エージェントに時間制限を設ける（1実験=最大15分等）
- **Dinekt**: AGCの15分制限ルールで対策済み

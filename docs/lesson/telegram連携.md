# Telegram連携

**カテゴリ**: lesson  
**登録日**: 2026-04-10 14:39:31  

---

- **Telegramプラグインの受信は途切れることがある** — セッション起動中・PC稼働中でも、grammyポーリングがメッセージを消費しつつClaude Code本体へのstdio配信が詰まる現象が発生。送信（reply）は正常でも受信だけ止まる。再起動が最短の回復手段（2026-04-02）
- **Cronキープアライブだけでは不十分** — 5分おきのCronジョブでセッションを起こしても受信途絶は解消しなかった。問題はセッションのアイドルではなくstdioパイプの詰まり
- **`.mcp.json` だけでは push 通知は届かない** — MCP サーバーのツールは使えるが、リアルタイム割り込みには `--channels server:<name>` フラグが必須。2層構造（`.mcp.json` + `--channels`）を理解していないと何度も同じ問題にハマる（2026-04-04）
- **再起動を求める前にセッションログを残す** — MCP 変更で再起動 → 記憶消失 → 同じ議論の繰り返しが3回発生。再起動前のログ保存を必須化（2026-04-04）

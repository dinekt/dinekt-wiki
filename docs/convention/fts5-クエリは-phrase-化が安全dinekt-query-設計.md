# FTS5 クエリは phrase 化が安全（dinekt-query 設計）

**カテゴリ**: convention  
**タグ**: fts5,cli,convention,2026-04-10  
**登録日**: 2026-04-10 14:59:11  

---

FTS5 のクエリ構文では '-' '"' '(' ')' '*' などが演算子になる。ユーザー入力をそのまま渡すとエラーになる（例: 'tech-news' は 'tech' 引く 'news' と解釈される）。

dinekt-query の対策:
1. デフォルトで全クエリをダブルクォートで phrase 化（fts_query = '"' + keyword.replace('"', '""') + '"'）
2. 利用者が明示的に FTS5 構文を使いたい場合は ':raw:' プレフィクスでオプトイン
3. これにより通常の検索は安全、上級ユーザーは高度な検索もできる
